## 텍스톰24 서비스 오픈 및 구조 업데이트
텍스톰24는 텍스트 마이닝 기술을 활용한 빅데이터 분석 솔루션으로, 텍스톰SV를 기반으로 제작되었습니다. 기존 기능을 개선하고 성능 최적화를 통해 서비스의 전반적인 기능을 향상시켰습니다.

이번 업데이트는 성능 최적화와 데이터 처리 효율성 향상에 중점을 두었습니다. 데이터 처리 속도를 개선하고, 새로운 데이터 소스와 분석 기법을 도입하여 사용자가 더 편리하고 만족스럽게 서비스를 이용할 수 있도록 하였습니다.


### 수정사항 리스트 📋

- **신규 크롤러 채널 추가 (RISS 국내 논문)**: RISS에서 제공하는 국내 논문 데이터를 추가로 크롤링하도록 시스템을 확장했습니다.
- **시계열 분석 추가**: 신규 데이터 형식 변경 및 과거 데이터 변환 기능을 추가하여 시계열 분석을 지원합니다.
- **Mecab 형태소 분석기 커스터마이징 + espressoK 제거**: 복합명사 처리 강화를 위해 espressoK를 제거하고 Mecab 분석기를 최적화했습니다.
- **코드 리펙토링 1 (분석 코드 통합)**: 5개 경로의 20개 분석 코드를 1개 경로의 8개 코드로 통합하여 코드의 효율성을 높였습니다.
- **코드 리펙토링 2 (메타데이터 관리 방식 변경)**: 분석 메타데이터 관리 방식을 DB 컬럼에서 JSON 파일로 변경하여 사용자 데이터 관리 효율성을 향상시켰습니다.
- **코드 리펙토링 3 (속도 최적화)**: for문을 iterrows, count, list comprehension 등으로 최적화하여 분석 코드 속도를 향상시켰습니다.
- **코드 리펙토링 4 (중복 모델 수정)**: for문 반복 횟수를 최소화하여 속도를 1.3~1.4배 향상시켰으며, 메모리 사용량이 증가하는 단점이 있습니다.

### 1. 신규 크롤러 채널 추가 (RISS 국내 논문) 📚

국내논문 수집기 RISS의 데이터 수집 추가 작업을 진행하였습니다. 이번 작업은 수집기 개선, 엘라스틱서치와의 연동, GCP 업데이트를 포함하며, 주요 사항은 다음과 같습니다.

#### 구현 과정 🔧

- **[구현 내용]**: 
  - **RISS 수집기 제작**: 신규 RISS 수집기를 제작하여 국내 논문 데이터를 수집할 수 있도록 했습니다.
  - **엘라스틱서치 인덱스 추가**: RISS 데이터의 효율적인 저장과 검색을 위해 엘라스틱서치에 인덱스를 새로 추가했습니다.
  - **DB 테이블 및 컬럼 변경**: RISS 데이터를 적절하게 저장하기 위해 DB 테이블 `keyword_riss`, `scraw_riss_datainfo`를 추가하고, 기존 테이블에 `riss`라는 `raw_type` 컬럼을 추가하여 RISS 데이터를 구분할 수 있도록 했습니다.

- **[기술 스택]**: 
  - **GCP (Google Cloud Platform)**: 클라우드 기반으로 데이터를 처리하고 업데이트했습니다.
  - **엘라스틱서치**: 데이터를 효율적으로 검색하고 관리하기 위해 사용했습니다.
  - **GRPC**: 수집기의 master-slave 구조 개선을 위해 사용되었습니다.
  - **MySQL**: 새로운 DB 테이블과 컬럼을 생성하고 관리했습니다.
  - **Python**: 크롤러 제작 및 데이터 처리 로직 구현에 사용되었습니다.

#### 테스트 및 결과값 📊

- **테스트 내용**: 
  - **수집기 테스트**: 229 및 경북클라우드 서버에서의 수집기 테스트를 진행하고, `textom-cube`로의 통합을 통해 일관된 데이터 포맷을 유지했습니다.
  - **GRPC 통신 테스트**: 개선된 master-slave 구조에서 데이터 전송이 제대로 이루어지는지 확인했습니다.

- **결과**: 
  - 기존 RISS 수집 시스템에서의 데이터 손실 문제와 처리 지연이 크게 개선되었습니다. 특히 날짜 데이터 추가 수집으로 RISS 데이터의 정확도가 향상되었습니다.
  - 새로운 **엘라스틱서치 인덱스**와 **DB 테이블**을 통해 RISS 데이터의 관리가 훨씬 용이해졌습니다.

- **성능 향상**:
  - **master-slave 구조** 개선을 통해 데이터 처리 속도와 안정성이 크게 향상되었습니다.
  - 날짜 데이터 추가로 RISS 데이터의 최신성 유지가 가능해졌습니다.
  - **GRPC 통신**을 통해 수집기 서버 간 데이터 전송이 효율적으로 이루어졌습니다.
